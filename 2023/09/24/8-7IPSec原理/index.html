8-7 IPSec原理

虚拟专用网 Virtual Private Network
建立公网上，由某一组织或某一群用户专用的通信网络

二层：L2TP和PPTP（基于PPP） 数据链路层
三层：IPSec和GRE 网络层
四层：SSL/TLS 传输层

实现数据加密的技术：PPTP、IPSec、SSL、TLS

L2TP和GRE不加密

实现虚拟专用网关键技术
1、隧道技术 ☆ 公司总部里面有文件服务器，在外出差，通过隧道访问公司内网文件
2、加解密技术 ☆
3、密钥管理技术
4、身份认证技术

虚拟专用解决方案
实现总分机构互联，常用IPSec
实现用户远程接入，比如出差拨号访问内网，常用SSL/TLS

二层隧道协议 PPTP和L2TP，都基于PPP协议。
PPTP（微软）只支持TCP/IP体系，网络层必须是IP协议。
L2TP可以运行在IP协议、X.25、帧中继、ATM网络上使用。

PPP协议包含链路控制协议LCP（Link Control Protocol） 和 网络控制协议NCP（NetWork Control Protocol）

PPP协议可以在点对点链路上传输多种上层协议的数据包，有校验位。
但HDLC只支持IP协议（网络层）

PPP认证方式：PAP和CHAP ☆

PAP：两次握手，口令以明文传送
CHAP：三次握手，传送的是HMAC散列值（哈希值、摘要值）
上层协议 地址协商 错误检测
PPP 多协议 能 可以
HDLC IP 不能 可以

IPSec基础

IPSec（IP Security）是IETF定义的一组协议，用于增强网络的安全性。

IPSec协议集提供如下安全服务：
1、数据完整性
2、认证
3、保密性
4、应用安全透明性

IPSec集成在IPv6中，给IPv4打补丁，增强安全性。

IPSec原理
AH：认证头
ESP：封装安全负荷 Encapsulating Security Payload
IKE：Internet密钥交换协议

IPSec协议 功能 代表协议
AH 数据完整性和源认证 MD5、SHA
ESP 数据加密 DES、3DES、AES
IKE 密钥生成和分发 DH

IPSec两种封装模式
原始报文 原来的IP头 + TCP + 数据
传输模式 原来的IP头 + AH + TCP + 数据
隧道模式 新的IP头 + AH + 原来的IP头 + TCP + 数据

IPSec是网络层协议
