华为配置专题11路由综合实验

RIP、OSPF、BGP配置

实验目的：
PC1可以ping通PC2，R1和R5要分别学到对端网段的路由

R1、R2、R3之间运行IGP
R3和R4之间是EBGP（不同AS之间）
R4和R5之间是IBGP

步骤

一、配置R1、R2、R3、R4、R5的接口ip
检查ip是否配错

二、配置路由

先配置AS 100里的，R1、R2、R3运行RIP协议

R1:
rip 1（进程号）
version 2 RIP2才支持子网掩码等
将3个网段都宣告进rip，只能宣告主类网络
network 100.0.0.0
network 12.0.0.0
network 13.0.0.0
quit

R2:
rip 1
version 2
network 12.0.0.0
network 23.0.0.0
quit

R3:
rip 1
version 2
network 23.0.0.0
network 13.0.0.0
quit
display ip routing-table

运行OSPF最短路径优先路由协议

R1、R2、R3删除rip协议配置
undo rip 1

R1
ospf 1（进程号）
area 0 （主干区域）
network 100.1.1.0 0.0.0.255（宣告网络）
network 12.1.1.0 0.0.0.3（/30就是3）
network 13.1.1.0 0.0.0.3
quit

R2
ospf 1
area 0
network 12.1.1.0 0.0.0.3
network 23.1.1.0 0.0.0.3

R3
ospf 1
area 0
network 23.1.1.1 0.0.0.3
network 13.1.1.0 0.0.0.3
display ospf peer brief 查看邻居关系是否建立，简写，收敛完成

运行IGP实现内部互通

配置BGP
R3
bgp 100 （R3在bgp 100里）
peer 34.1.1.2 as-number 200 对等体

R4
bgp 200
peer 34.1.1.1 as-number 100
peer 45.1.1.2 as-number 200

R5
bgp 200
peer 45.1.1.1 as-number 200

undo peer 34.1.1.1 删除指定ip的peer

display bgp peer 看bgp邻居关系

bgp路由比较特别，要手工的宣告或者把其他路由协议学到的路由导入到bgp里面才有，只有邻居关系bgp是没有路由的

2种方式

①在bgp里把路由宣告进去
比如在R5里
ipv4-family unicast ipv4的单播路由
network 200.1.1.0 24 把pc2的网段宣告进bgp，这时R3和R4都学到了PC2网段的路由

②路由的引入

pc1 ping通 pc2

R3已经学到了200.1.1.0/24的路由，把R3通过BGP学到的路由引入到OSPF进程里
R3右边通过BGP学到了200的路由，右边学到的路由导入到左边来就行

互导 叫 路由的引入
bgp的路由引入到ospf里

R3上
ospf 1
import-route bgp

ospf路由引入到bgp里

R3上
bgp 100
import-route ospf 1

这时R4上有100.1.1.0/24的路由，但R5上没有

display bgp routing-table

R5上已经学到了左边OSPF的路由，但没有加表

因为R5没有34.1.1.0/30的路由，就不会加入路由表
下一跳不可达

1、在R5上写静态路由，这样就加表了，让下一跳可达

ip route-static 34.1.1.0 30 45.1.1.1

2、
删除上面配置的静态路由

在R4上配置

bgp 200

peer 45.1.1.2 next-hop-local R4向R5 bgp peer宣告路由时，把下一跳设置成自己

先前是从左边学到的路由，对外宣告的时候，下一跳还是R3，R5到R3不可达，所以宣告过去路由，R5不会加表

如果下一跳改成自己，R4和R5是直连路由，直接可以通，所以就加表

R5上
display ip routing-table

display bgp routing-table

pc1
ping 200.1.1.1 可以通
tracert 200.1.1.1

把R1到R3之间线路关闭

R1上
interface GigabitEthernet 0/0/2
shutdown

再在pc1上tracert 200.1.1.1
