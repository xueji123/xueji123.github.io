华为配置专题10浮动路由和BFD配置

1、先把router 4个接口的ip配置上

display ip interface brief 查看是否配置错误

CMCC、CUCC、internet 3台设备的接口也都配置好

interface LoopBack 0 环回口一般用来做测试，比较稳定

2、

CMCC、CUCC、internet 3台设备 配置OSPF，把所有的网段宣告出去

ospf 1
area 0
network 0.0.0.0 0.0.0.0 把自己直连的所有网段通告进OSPF中 （传统的方法是把当前路由器直连的网段宣告进OSPF）

display ospf peer brief 查看邻居路由

3、配置出口路由器router

NAT配置
easy ip 把所有内部ip都转成出口ip地址

acl 2000
rule 10 permit source 192.168.10.0 0.0.0.255
rule 20 permit source 192.168.20.0 0.0.0.255
quit

interface GigabitEthernet 0/0/2
nat outbound 2000
quit

interface GigabitEthernet 4/0/0
nat outbound 2000
quit

配置了2个NAT，到底往哪边走，配置出口路由决定

配置2条出口路由（2个运营商 电信和联通），2条等价（优先级一样的话）的默认路由，出口必须要用默认路由

ip route-static 0.0.0.0 0 12.1.1.2

ip route-static 0.0.0.0 0 13.1.1.2

在PC1和PC2 tracert 互联网地址 22.22.22.22，发现都是从 13.1.1.2走的

所以要在router配置负载均衡 load-balance src-ip

还是不行，是不是上面链路不通，可以把上面关闭试试看
interface GigabitEthernet 4/0/0
shutdown

取消关闭接口
undo shutdown
display ip interface brief 查看接口状态

还是只走下面的13.1.1.2，因为华为的AR中低端系列路由器，不是基于单个包进行负载均衡的，不能设置。
高端NE系列，NE40、NE20是支持的。基于单包做负载均衡，对路由器的CPU和缓存消耗较大

出口router上调节路由的Preference，越小越优先
ip route-static 0.0.0.0 0 13.1.1.2 preference 100

display ip routing-table 查看联通路由已经没有了

这就是浮动路由！！！
把preference调大后，就不优先了，不优先就隐藏起来
上面电信的路由一挂，就会走下面的preference 100的路由，会重新启用

interface GigabitEthernet 0/0/2
shutdown
quit

display ip routing-table

电信的 0/0/0挂了，也会走下面preference 100的路由
在CTCC上 interface GigabitEthernet 0/0/0
shutdown
quit

在router和CTCC之间增加一台交换机，并把CTCC的 0/0/0接口shutdown

这时候出口路由还是走CTCC，这时候需要通过BFD来解决该问题（router探测不到LSW1和CTCC之间线路故障）

BFD是Bidirectional Forwarding Detection的缩写，它是一个用于检测两个转发点之间故障的网络协议
https://info.support.huawei.com/info-finder/encyclopedia/zh/BFD.html

BFD链路质量、接口地址探测

display bfd session all 查看BFD状态

模拟器不支持单臂回声，因此要在2台路由器设备都配置BFD才行

router上：
bfd
quit
bfd 1 bind peer-ip 12.1.1.2 source-ip 12.1.1.1 auto
commit 让BFD生效

display bfd session all

这时去router上把 0/0/2 shutdown
在CTCC上监测BFD就会是 Init关闭状态了

这时在router上做一件事情，上面这条默认路由跟踪一个BFD
ip route-static 0.0.0.0 12.1.1.2 track bfd-session 1
如果这个bfd不通了后，这条路由就会删掉

在CTCC上
interface GigabitEthernet 0/0/0
shutdown

display bfd session all

router上的电信路由就被删除了
display ip routing-table

浮动路由和BFD联合应用！！！
